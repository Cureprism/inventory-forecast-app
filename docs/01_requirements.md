# 要件定義書

## 1. 背景
在庫を紙や簡易的な記録で管理していると、更新が後回しになったり、
実際の棚と記録がズレてしまうことが起きやすい。履歴が増えるほど
集計や確認に手間がかかり、今の在庫状況や補充タイミングを判断しにくくなる。
この課題を解消するため、商品登録・入出庫管理・簡易的な在庫予測を一つの画面で
行えるWebアプリを開発する。

## 2. 目的
本システムは、商品ごとの在庫を分かりやすく把握できるようにし、入出庫の履歴を
もとに現在庫を自動で算出する。さらに、直近の出庫傾向から在庫の推移を予測し、
補充のタイミングを判断しやすくすることを目的とする。

## 3. 想定ユーザー
- 小規模小売業・個人事業主  
- 倉庫・仕入れ管理担当者

## 4. 機能一覧
| 機能区分 | 機能名 | 概要 |
|----------|---------|------|
| 商品管理 | 商品登録・編集・削除 | 商品名や管理番号などの基本情報を管理 |
| 入出庫管理 | 入庫・出庫登録 | 商品ごとの入出庫履歴を登録・表示 |
| 在庫管理 | 在庫数表示 | 入出庫の差分から現在庫を自動で算出 |
| 在庫予測 | 在庫推移の予測 | 直近の出庫傾向をもとに在庫の推移を予測 |
| 表示補助 | 在庫数の強調表示 | 安全在庫を下回る場合、在庫数を赤字で表示 |

## 5. 非機能要件
| 項目 | 内容 |
|------|------|
| 開発環境 | Laravel 12 / PHP 8.3 / MySQL 8 |
| データ規模 | 数百件程度の履歴データを想定 |
| 性能 | 在庫一覧・予測処理がストレスなく表示されること |
| セキュリティ | CSRF対策・入力バリデーションを実施 |
| 運用環境 | Heroku 上で公開する閲覧用の環境を想定 |
| 保守性 | Laravel標準構成（MVC）に沿い、小規模な拡張に対応 |
| 認証 | ログイン機能を備え、閲覧用アカウントをログイン画面で案内 |
| 対応環境 | PCブラウザでの閲覧を想定（スマホは簡易対応） |

## 6. システム構成
- フロントエンド：Blade テンプレート（Tailwind CSS を一部使用）
- サーバーサイド：Laravel（Controller / Request / Service）
- データベース：MySQL
- 予測処理：サービスクラスで出庫履歴をもとに在庫推移を算出（移動平均）
